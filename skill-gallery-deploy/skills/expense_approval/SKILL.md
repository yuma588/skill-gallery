# 费用审批

## 概述

本技能支持系统化处理费用报销申请，并根据公司标准和差旅政策进行验证。它提供了结构化的工作流程，用于审核费用、检查与预定义限额的合规性，并生成审批建议。

## 快速开始

处理费用报销申请的步骤：

1. 审核费用详情（金额、类别、日期、收据）
2. 使用 `standards/expense_limits.json` 检查适用的限额
3. 验证是否符合 `policies/travel_policy.md` 中的相关政策
4. 生成带有理由的审批建议

## 费用限额

通过 `standards/expense_limits.json` 访问公司费用限额，包含：

- 餐饮、交通和住宿的每日津贴
- 按类别划分的支出限额
- 基于金额的分层审批阈值
- 住宿标准的城市分类

验证费用时：
1. 将 `standards/expense_limits.json` 加载到上下文中
2. 将费用类别匹配到相关的限额部分
3. 比较提交金额与定义的限额
4. 记录任何超额或例外情况

## 差旅政策

参考 `policies/travel_policy.md` 中的公司差旅政策，涵盖：

- 预订要求和提前通知期
- 批准的预订渠道和平台
- 交通标准（航班舱位、火车座位）
- 住宿指南（酒店等级、城市分类）
- 餐饮津贴规则和文档要求
- 紧急费用处理程序
- 报销所需的支持文档

应用差旅政策时：
1. 将 `policies/travel_policy.md` 加载到上下文中
2. 识别与费用类型相关的政策部分
3. 验证所有政策要求是否满足
4. 记录任何政策违规或缺失文档

## 审批工作流

### 步骤1：初步审核

检查费用提交的完整性：
- 金额与收据匹配
- 类别分类正确
- 日期与业务目的相符
- 所需文档齐全

### 步骤2：限额验证

根据 `standards/expense_limits.json` 中的适用限额进行验证：
- 检查每日/交易限额
- 验证累计总额（如适用）
- 根据金额确定审批级别

### 步骤3：政策合规性

验证是否符合 `policies/travel_policy.md` 中的相关政策：
- 预订提前要求
- 首选供应商使用
- 文档完整性
- 政策例外理由

### 步骤4：决策

生成审批建议：
- **批准**：所有限额和政策均满足
- **带备注批准**：需要澄清的轻微问题
- **拒绝**：政策违规或文档缺失
- **升级**：超出审批权限

### 步骤5：文档记录

提供清晰的决策理由：
- 引用具体的限额或政策条款
- 解释任何偏差或例外
- 注明所需的后续行动

## 输出格式

按照以下结构生成审批摘要：

```
费用审批摘要
========================

提交详情：
- 员工：[姓名]
- 提交日期：[日期]
- 总金额：[金额]
- 类别：[类别]

验证结果：
- 限额合规性：[合规/不合规]
- 政策合规性：[合规/不合规]
- 所需审批：[审批级别]

决策：[批准/拒绝/升级]

理由：
[详细解释，引用具体限额和政策]

后续行动：
[如适用]
```

## 常见场景

### 日常费用报销

餐饮和交通报销：
- 根据 `expense_limits.json` 验证每日总额与累计限额
- 检查 `travel_policy.md` 中的收据要求
- 验证业务目的文档

### 多日差旅

延长商务旅行：
- 计算所有日期的总费用
- 验证每个类别的每日限额
- 按城市分类检查住宿标准
- 确认遵守预订提前政策

### 紧急费用

例外报销：
- 参考 `travel_policy.md` 中的紧急处理程序
- 验证紧急情况证明文档
- 检查是否获得预先批准（如需要）
- 记录例外理由

## 资源

本技能包含结构化参考文档：

### standards/expense_limits.json

定义公司费用限额和阈值，包括：
- 餐饮津贴（早餐、午餐、晚餐、每日总额）
- 交通限额（出租车、公共交通、租车）
- 按城市等级划分的住宿标准
- 杂项费用类别
- 审批级别阈值

**适用场景：** 实时限额验证、金额核实、审批级别确定

### policies/travel_policy.md

包含全面的差旅费用政策，涵盖：
- 预订要求和提前通知期
- 批准的预订渠道和供应商列表
- 交通舱位和座位标准
- 住宿指南和酒店分类
- 餐饮津贴规则和发票要求
- 紧急费用处理和例外流程
- 报销提交所需的文档

**适用场景：** 政策合规性验证、例外处理、文档要求
