# 空白页面故障排查指南

## 快速诊断步骤

### 1. 检查浏览器控制台
按 `F12` 打开开发者工具，查看 Console 标签页是否有错误信息。

### 2. 检查网络请求
在 Network 标签页中，查看是否有红色的失败请求。

### 3. 确认开发服务器是否运行
在终端中应该看到类似以下输出：
```
VITE v5.x.x  ready in xxx ms

➜  Local:   http://localhost:3000/
```

---

## 常见问题及解决方案

### 问题 1：页面完全空白，没有任何内容

**可能原因**：
- JavaScript 错误导致应用崩溃
- Vue 未正确挂载

**解决方案**：
1. 打开浏览器控制台（F12），查看 Console 中的错误信息
2. 如果看到 "Cannot find module" 或 "module not found"，说明依赖未安装
3. 运行 `npm install` 重新安装依赖

### 问题 2：页面有布局但无内容

**可能原因**：
- API 请求失败
- 数据加载错误

**解决方案**：
1. 查看控制台是否有 "使用模拟数据" 的日志
2. 查看是否有网络请求错误（Network 标签）
3. 现已添加模拟数据，即使 API 失败也能显示内容

### 问题 3：端口访问被拒绝

**可能原因**：
- 端口被占用
- 防火墙阻止

**解决方案**：
1. 检查 3000 端口是否被占用：`netstat -ano | findstr :3000`
2. 修改 `vite.config.ts` 中的端口：
   ```typescript
   server: {
     port: 5173,  // 改成其他端口
   }
   ```

### 问题 4：样式不显示

**可能原因**：
- Sass 编译错误
- CSS 文件路径错误

**解决方案**：
1. 检查控制台是否有 CSS 相关错误
2. 清除浏览器缓存：`Ctrl + Shift + R`
